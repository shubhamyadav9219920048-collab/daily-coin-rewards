What is Daily Rewards Pro?

Daily Rewards Pro is an advanced promotional rewards platform built around a secure digital coin ledger. By engaging with daily login calendars and invite programs, users accumulate loyalty points in the form of virtual coins. Our infrastructure guarantees ledger accuracy and syncs immediately to real-time cloud Firestore databases.

How Do I Earn Coins?

Earning is simple, linear, and completely free. Standard users can tap the Daily Coin to instantly stack +10 coins every 24 hours. Additionally, expanding your network via our referral engine rewards both the invitee and invited partner. (Rewarded video ad features are currently coming soon pending Google publisher audits).

How Do Referrals Work?

Once logged in, you receive a unique, server-generated 6-character referral code and sharing link. When a new user signs up or claims your invite code, our batch-transaction system immediately distributes +50 coins to your ledger AND +50 coins to the invitee's balance. There are no referral networks depth caps!

How Withdrawals Work

Once your ledger reaches 1000 coins (equivalent to ₹10 payout value), you can request payout directly to any registered UPI ID. Payout requests do not immediately deduct from your ledger; they enter a verification queue. Certified administrators review account activity logs for strict compliance before processing.

Withdrawal & Payout Policy

Read about our secure coin redemptions, verification queues, and processing times.

Daily Rewards Pro takes secure rewards tracking seriously. To prevent fraudulent botting, multi-accounting, and system abuse, all withdrawal requests are governed by this strict **Withdrawal Policy**. By submitting a payout request, you fully agree to these guidelines.

1. Conversion Rates & Limits

Our virtual coins are promotional loyalty points that can be redeemed for UPI cash payouts subject to administrative verification:

  • Conversion Ratio: 100 virtual coins = ₹1 INR.
  • Minimum Payout Threshold: 1,000 coins (equivalent to ₹10). You cannot request a withdrawal for any amount below this limit.
  • Daily Limits: Users may submit up to 1 payout request per 24 hours. The maximum daily redemption limit is 10,000 coins (₹100) to ensure pool sustainability.

2. Step-by-Step Payout Process

To successfully claim your rewards, follow these official steps:

  1. Reach a minimum balance of 1,000 coins inside your secure ledger.
  2. Navigate to the Withdrawal Portal tab on your dashboard.
  3. Enter your valid, personal UPI ID (e.g., name@okaxis, phone@paytm) and the number of coins to redeem.
  4. Tap the Request Payout button. The coins will temporarily enter the "Pending" status and undergo compliance review.

3. Secure Manual Verification Queue

To comply with publisher guidelines and maintain advertiser trust, **no payouts are processed automatically**.

Each request is queued for manual analysis by our administrator team. Our secure review includes auditing:

  • Login frequency logs and claim schedules (detection of auto-clickers or macros).
  • IP addresses (detection of VPNs, proxies, or Tor nodes).
  • Referral networks logs (ensuring referred friends are legitimate unique devices).
  • Device signatures (confirming only 1 account per physical mobile device).

The standard processing time is 24 to 48 business hours. You can monitor the real-time status (Pending, Approved, Rejected) directly under your Withdrawal History.

4. Grounds for Immediate Forfeiture and Ban

If an administrator detects any of the following during review, the payout request will be immediately REJECTED, the account will be permanently banned from the database, and any accumulated points will be forfeited:

  • Using emulator software, script injectors, virtual operating systems, or cloning apps.
  • Falsifying UPI IDs, registering multiple user accounts on the same device, or using temporary/disposable emails.
  • Using software to intercept or manipulate the Firebase client-side request packets.
  • Sponsor ad blocking, ad tampering, or script automation.

Database Settings

By default, the app runs in local **Simulated Sandbox mode**. If you want to connect real cloud storage, paste your web app's Firebase credentials below and click Save.

Firestore Rule Requirement
Ensure your database has both Users and Withdrawals collections configured. Example security rules: 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /Users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /Withdrawals/{reqId} {
      allow read, write: if request.auth != null;
    }
  }
}
